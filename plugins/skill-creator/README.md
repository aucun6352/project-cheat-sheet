# Skill Creator Plugin

Claude Code를 위한 대화형 스킬 생성 플러그인입니다.

## 📖 개요

이 플러그인은 Claude Code에서 사용할 수 있는 스킬(Skill)을 **지능형으로** 생성할 수 있도록 도와줍니다.

### ✨ 주요 특징

- 🧠 **대화 내용 분석**: 현재 세션의 대화를 자동으로 분석하여 스킬 정보 추출
- 💡 **지능형 추천**: 분석 결과를 바탕으로 스킬 이름, 설명, 내용을 자동으로 제안
- ✏️ **선택형 입력**: 추천된 내용 중 선택하거나 직접 수정 가능
- 📝 **표준화된 구조**: 일관된 품질의 SKILL.md 자동 생성

복잡한 파일 구조를 직접 만들 필요 없이, 대화만 하면 자동으로 스킬이 생성됩니다!

## 🚀 사용법

### 스킬 생성하기

```bash
/create-skill
```

커맨드를 실행하면 다음과 같이 진행됩니다:

#### 1️⃣ 대화 분석 (자동, 조건부)
Claude가 현재 세션의 대화 내용을 자동으로 분석합니다:
- 📊 주요 주제와 키워드 식별
- 💻 코드 예제 추출
- 📌 핵심 개념과 패턴 파악
- ✅ 모범 사례 수집
- ⚠️ 주의사항 식별

**⚠️ 대화 내용이 부족하면 이 단계는 자동으로 건너뜁니다!**
- 메시지 5개 미만 → 직접 입력 모드
- 기술적 내용 없음 → 직접 입력 모드

#### 2️⃣ 정보 수집 (추천형 또는 직접 입력형)

**대화 분석을 완료한 경우 (추천 모드)**:
1. **스킬 이름**: 2-3개의 추천 이름 중 선택하거나 직접 입력
   - 예: `skill-creator-workflow`, `interactive-skill-generator`
2. **스킬 설명**: 자동 생성된 설명을 사용하거나 수정
3. **사용 시나리오**: 대화에서 추출한 시나리오 확인 및 추가
4. **핵심 개념**: 정리된 개념을 사용하거나 보완
5. **패턴/예제**: 발견된 코드 예제 선택 또는 추가
6. **모범 사례**: 수집된 권장사항 확인 및 추가
7. **주의사항**: 식별된 함정과 경고 확인 및 추가

**대화 분석을 건너뛴 경우 (직접 입력 모드)**:
- 모든 항목을 직접 입력
- 예시와 안내가 제공됨

#### 3️⃣ 스킬 생성 (자동)
선택/입력한 내용을 바탕으로 표준 SKILL.md 파일을 자동 생성합니다!

### 생성된 스킬 구조

스킬은 **프로젝트 루트 디렉토리**에 생성됩니다:

```
{프로젝트-루트}/
└── {스킬-이름}/
    └── SKILL.md              # 메인 스킬 문서
```

### SKILL.md 구조

생성되는 스킬 파일은 다음과 같은 표준 구조를 따릅니다:

```markdown
---
name: skill-name
description: 스킬에 대한 간단한 설명
---

# 스킬 제목

## When to Use This Skill
- 사용 시나리오 1
- 사용 시나리오 2
- ...

## Core Concepts
핵심 개념에 대한 설명

## Patterns
### Pattern 1: 패턴 이름
```language
// 코드 예제
```

## Best Practices
- 모범 사례 1
- 모범 사례 2
- ...

## Common Pitfalls
- 주의사항 1
- 주의사항 2
- ...
```

└── commands/
    └── create-skill.md       # 스킬 생성 커맨드
```

### 생성된 스킬 위치
스킬은 프로젝트 루트에 생성됩니다:
```
{프로젝트-루트}/
├── plugins/
│   └── skill-creator/        # 플러그인 디렉토리
├── {스킬-이름-1}/            # 생성된 스킬 1
│   └── SKILL.md
├── {스킬-이름-2}/            # 생성된 스킬 2
│   └── SKILL.md
└── ...
```

## 💡 예제

### 예제 1: 대화 기반 자동 생성

**시나리오**: API 디자인에 대해 Claude와 30분간 대화한 후

```bash
/create-skill
```

**Claude의 분석 결과**:
```
📊 대화 분석 완료!

🔍 발견된 내용:
- 주요 주제: RESTful API 설계, 엔드포인트 구조, 에러 처리
- 코드 예제: 5개 (Express.js, HTTP 상태 코드)
- 논의된 패턴: REST 리소스 모델링, 버전 관리
- 핵심 개념: 3개 (리소스 중심 설계, 일관성, 상태 코드)

💡 이제 추천 내용을 바탕으로 스킬을 생성합니다.
```

**추천된 스킬 이름**:
1. `api-design-principles` ✅ (선택)
2. `restful-api-patterns`
3. `api-error-handling`

**자동 생성된 설명**:
> "RESTful API 설계 원칙과 모범 사례를 제공하며, 일관된 엔드포인트 구조와 적절한 에러 처리 방법을 안내합니다."

**추출된 사용 시나리오**:
- ✅ 새로운 API 엔드포인트를 설계할 때
- ✅ API 구조를 리뷰할 때
- ✅ 에러 응답 형식을 표준화할 때

→ 몇 번의 클릭만으로 스킬 생성 완료!

### 예제 2: 빈 대화에서 수동 생성

**시나리오**: 새로운 세션에서 처음부터 스킬 생성

```bash
/create-skill
```

**Claude의 분석 결과**:
```
📊 대화 내용 분석 결과

ℹ️ 현재 세션의 대화 내용이 부족하여 자동 추천이 어렵습니다.
   (대화 메시지: 1개, 기술적 내용: 없음)

💡 직접 입력 모드로 진행합니다.
   - 모든 정보를 수동으로 입력해주세요
   - 또는 스킬 주제에 대해 먼저 대화를 나눈 후 다시 시도해보세요
```

**진행 방식**:
- Phase 0 (대화 분석) 자동으로 건너뜀 ⏭️
- Phase 1부터 직접 입력 모드로 진행
- 모든 질문에 수동으로 답변 입력

→ 추천 없이 모든 정보를 직접 입력하는 방식으로 진행

## 🎯 스킬 활용 방법

생성된 스킬은 Claude Code와의 대화에서 다음과 같이 활용할 수 있습니다:

1. **참조로 활용**: `@{스킬-이름}/SKILL.md`를 통해 스킬 문서를 컨텍스트에 포함
2. **가이드로 사용**: 특정 작업을 수행할 때 스킬의 모범 사례 참고
3. **템플릿으로 활용**: 스킬에 포함된 코드 예제를 프로젝트에 적용

**예시**:
```
"API를 설계할 때 모범 사례를 알려줘 @api-design-principles/SKILL.md"
```

## 🔧 커스터마이징

생성된 스킬은 언제든지 수정할 수 있습니다:

1. 프로젝트 루트의 `{스킬-이름}/SKILL.md` 파일을 열기
2. 필요한 섹션 추가 또는 수정
3. 저장 후 즉시 사용 가능

## 📚 추가 리소스

스킬 디렉토리 내에 추가 리소스를 구성할 수 있습니다:

```
{스킬-이름}/
├── SKILL.md                    # 메인 문서
├── assets/                     # 코드 템플릿, 체크리스트
│   ├── template.ts
│   └── checklist.md
└── references/                 # 심화 가이드, 참고 문서
    └── advanced-guide.md
```

- 스킬에 코드 템플릿이나 체크리스트를 추가하려면 `{스킬-이름}/assets/` 디렉토리 생성
- 심화 가이드나 참고 문서는 `{스킬-이름}/references/` 디렉토리에 추가
- 각 리소스 파일은 SKILL.md의 "Resources" 섹션에서 링크

## 🤝 기여

새로운 스킬 템플릿이나 개선 사항이 있다면:

1. 스킬을 생성하고 테스트
2. 유용한 패턴이나 모범 사례 문서화
3. 팀과 공유
